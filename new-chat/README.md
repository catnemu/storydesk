# Story Desk

小説を書くためのPWAです。章、本文、設定メモを端末内に自動保存し、iPhoneのSafariから「ホーム画面に追加」して単体アプリのように使えます。

## 保存の仕組み

本文データはサーバーには送信されません。GitHub PagesやNetlifyにはアプリ本体だけを置き、原稿は開いている端末のブラウザ内に保存します。

保存先は二重化しています。

- `localStorage`: すばやく読み書きするための保存先
- `IndexedDB`: localStorageとは別の端末内データベース

入力すると自動保存されます。画面には `保存中...` から `保存済み（端末内）` のように表示されます。

既存データは起動時に自動マイグレーションされます。古いバックアップや古い端末内データに各話の `status` や `updatedAt` がなくても、下書きステータスと更新日時を補完して読み込みます。

## 作品インデックス

上部の `目次` ボタンで、作品インデックスを表示します。

インデックスでは最大30話まで管理できます。各話について、タイトル、文字数、最終更新日、ステータス（下書き / 清書 / 完成）を一覧できます。各行の `編集` またはタイトルを押すと、その話の編集画面へ移動します。

作品全体の合計文字数もインデックス上部に表示されます。

## 作品一覧

上部の `作品` ボタンで、端末内に保存されている作品一覧を表示します。

旧バージョンの単一作品データは、初回起動時に作品一覧の1件目として自動マイグレーションされます。作品一覧から `開く` を押すと、その作品の編集画面に移動します。

左上の `+` ボタンで新しい作品を追加できます。新規作品を追加しても既存作品は削除されません。

## バックアップ

上部の `TXT` ボタンで、全話を1つのテキストファイルとして書き出します。

上部の `JSON` ボタンで、先に現在開いている作品の全話TXTを書き出し、その後に全作品を含む復元用JSONを書き出します。安全のため、長く書いた日はJSONバックアップを残してください。

上部の `復元` ボタンで、JSONバックアップを読み込んで端末内データを復元できます。復元すると現在の端末内データは選んだJSONの内容に置き換わります。

新規作品を作る時も、現在の作品を置き換える前にTXTとJSONを書き出します。

## NetlifyやGitHub Pagesの再デプロイ

同じ公開URLのまま再デプロイする限り、通常は `localStorage` と `IndexedDB` の本文データは消えません。再デプロイで置き換わるのはアプリ本体のHTML/CSS/JSです。

ただし、公開URLの origin が変わると別アプリ扱いになり、前の端末内データは新しいURLから見えません。

例:

- `https://example.netlify.app/` から `https://example.netlify.app/` への再デプロイ: 通常は消えない
- `https://old-name.netlify.app/` から `https://new-name.netlify.app/` へ変更: 別保存領域になる
- `https://user.github.io/repo-a/` から `https://user.github.io/repo-b/` へ変更: 別保存領域になる

## データが消える可能性がある操作

次の操作では、端末内の保存データが消える、または見えなくなる可能性があります。

- Safariやブラウザの「履歴とWebサイトデータを消去」
- iPhoneのストレージ整理やブラウザデータ削除
- プライベートブラウズでの利用
- 公開URL、ドメイン、GitHub Pagesのリポジトリ名、Netlifyのサイト名を変える
- 別のブラウザや別の端末で開く
- PWAを削除したうえで、関連するWebサイトデータも削除する
- アプリ内のJSON復元で別データに置き換える

安心して使うために、重要な作業後は `JSON` ボタンでTXTとJSONを両方残してください。

## GitHub Pagesで公開

1. GitHubで新しいリポジトリを作る
2. このフォルダのファイルをリポジトリに入れて `main` ブランチへpushする
3. GitHubのリポジトリ画面で `Settings` -> `Pages` を開く
4. `Build and deployment` の `Source` を `GitHub Actions` にする
5. Actionsのデプロイ完了後、表示されたPages URLを開く

`.github/workflows/pages.yml` が入っているので、GitHub Pagesは静的ファイルをそのまま公開します。

## Netlifyで公開

1. Netlifyで `Add new site` -> `Import an existing project` を選ぶ
2. このリポジトリを選ぶ
3. Build commandは空、Publish directoryは `.` のままデプロイする

`netlify.toml` が入っているので、service workerとmanifest向けのヘッダーも設定されます。

## iPhoneでホーム画面に追加

1. iPhoneのSafariで公開URLを開く
2. 共有ボタンを押す
3. `ホーム画面に追加` を選ぶ
4. `追加` を押す

ホーム画面版も同じURLの端末内データを使います。更新が反映されない時は、Safariで公開URLを開いて再読み込みしてから、ホーム画面版を開き直してください。
